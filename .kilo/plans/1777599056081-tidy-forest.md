## 2. Diagnóstico: O que NÃO está pronto (Gaps críticos)

### 2.1 GAP CRÍTICO — Tipagem Supabase inconsistente
- `src/types/supabase.ts` contém tipos de um sistema legado completo (blogs, articles, profiles, tags, article_jobs) que **não pertencem** ao FaceStore
- As entidades do commerce (`partners`, `products`, `clicks`) existem nos tipos mas misturadas com `categories` que tem `blog_id` (legado)
- **Impacto:** componentes de commerce tipam `Product` com join `partners` que funciona no mock mas pode quebrar com dados reais

### 2.2 GAP CRÍTICO — Migration vs Schema real do Supabase
- A migration `20260212_initial_commerce_schema.sql` pode ou não ter sido aplicada ao banco remoto
- A tipagem gerada sugere que o banco remoto tem tabelas do sistema legado (blogs, articles, profiles) que não estão na migration
- **Impacto:** não sabemos se o schema de commerce está efetivamente criado no banco

### 2.3 GAP ALTO — Dados reais inexistentes
- Nenhum seed data. O catálogo está vazio.
- Sem admin ou interface de gestão de catálogo
- **Impacto:** o sistema não gera valor sem dados de parceiros/produtos reais

### 2.4 GAP ALTO — CTAs de tracking não conectados
- Os `ProductCard`, `ReviewBox` e `ComparisonTable` linkam direto para `affiliate_link`
- Não usam `/api/track?pid=ID&slug=SLUG` para rastrear cliques
- **Impacto:** cliques saem sem rastreabilidade

### 2.5 GAP MÉDIO — Header sem menu mobile
- Header só mostra links em `md:` (desktop). Não há hamburger menu ou drawer para mobile
- **Impacto:** experiência mobile quebrada na navegação

### 2.6 GAP MÉDIO — Links vazios no Footer
- "Cases de Sucesso", "Fale Conosco", "Termos de Uso", "Privacidade" apontam para `#`
- **Impacto:** navegação incompleta

### 2.7 GAP MÉDIO — SEO e metadata mínimos
- Layout com metadata genérica ("FaceStore — Facebrasil Commerce Hub")
- Sem Open Graph, sem favicon customizado, sem sitemap
- **Impacto:** descoberta orgânica inexistente

### 2.8 GAP BAIXO — Pasta `src/components/ui/` vazia
- Componentes de UI reutilizáveis (Badge, Button, etc.) não foram extraídos
- **Impacto:** refactoring futuro necessário, não bloqueia operação

## 3. Plano de Ação — Caminho mais rápido para operacionalizar

| # | Ação | Detalhe | Prioridade |

## 5. Decisões pendentes que bloqueiam execução

1. **O banco Supabase remoto tem o schema de commerce aplicado?** Se não, aplicar a migration é a primeira ação.
2. **Tipos legados (blogs, articles, profiles) são necessários?** Se não, remover simplifica drasticamente.
3. **O sistema de admin será Supabase Studio ou uma interface customizada?** Para velocidade, Studio é suficiente.
4. **Qual domínio para deploy?** Necessário para configurar na Vercel.
